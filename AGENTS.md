## 4. Standing Rules — Iron-Forged

These extend (and where noted, take exception to) the master AGENTS.md rules. All other master rules — accuracy, three-phase code review, no file deletion, living docs, dev-port assignment, communication norms — apply unchanged.

### Stack exception (Code rule)

The master AGENTS.md mandates **Python exclusively** where viable. **This project is an established TypeScript/Astro/React stack** built before workspace registration. The deviation is explicit and acknowledged: keep all production code in TypeScript/Astro; use Node ESM (`*.mjs`) for utility scripts in `scripts/`; reach for Python only when the task is genuinely better served by it (image conversion / PDF generation are existing examples — both kept in `C:\Users\mathe\Downloads\heic-thumbs\` so they don't pollute the project tree).

### Codex owns git on this project

Commit + push per logical chunk during a working session — Codex does not ask before committing or pushing. Branch is `main`; remote is `https://github.com/CRRrvPark/CRR-RV-Park-Website`. Commit messages follow the existing project style (subject + paragraph body explaining *why* + a `Co-Authored-By:` trailer). This is a workflow Mathew explicitly established and reaffirmed across many sessions; do not re-ask.

*(Source: prior session memory `workflow_claude_owns_git.md`.)*

### Maps / Places / content curation is conservative by default

When backfilling lat/lng, Google Place IDs, cached Google data, photos, etc., **only fill rows that already exist** in `things_to_do`, `local_places`, `trails`, and `park_sites`. Do not INSERT new rows without explicit owner approval. Cover everything currently listed, but never auto-expand. Exception cases (where Mathew has said "go add this") get a one-line callout in the relevant script.

*(Source: prior session memory `feedback_maps_curation_conservative.md`.)*

### Image selection — no lazy reuse, no misleading subjects

Each card / hero / popover gets a **distinct** image. Reusing the same shot across multiple cards is a trust violation. When a thematic stock image is needed (a "concept of dining" card with no specific Place to draw from), pick from verified Unsplash photo IDs — fetch the parent page on `unsplash.com` to confirm the subject before committing the URL. A generic close-up of the subject beats misleading scenery. Hold this bar at item 50 the same as at item 5.

*(Source: prior session memory `feedback_image_quality_bar.md`.)*

### Living docs reflex (master §4 *Living documentation*) — what to update on this project specifically

When you change…
- a **public page or layout** → update or check `HANDOFF-V5-PRODUCTION-REMODEL.md`, `PROJECT-DETAILS.md`, and `NETLIFY-AI-EDITING.md` if the change affects the active production model.
- the **stack version, deps, ports, env vars** → update `RUNBOOK.md`, `SETUP.md`, `.env.example`, and the relevant section of this charter.
- a **migration** → migration goes in `supabase/migrations/NNN_*.sql`; run `node scripts/supabase-migrate.mjs`; mention in `RUNBOOK.md` if it changes deployment.
- **anything in this charter** → bump *Last updated*, add a Revision History row, then run `py "C:\Codex Workspace\split_charter.py"` to refresh the Grok split.

## 5. Environment, Stack, and Operational Snapshot

**Stack:** Astro 5.18 (SSR via Netlify adapter + selective `prerender = true` for static routes) · TypeScript · React islands for operational admin/interactive surfaces · Supabase (Postgres + Auth + Storage) · Netlify (hosting, functions, scheduled functions, Deploy Previews, Agent Runners) · Zoho WorkDrive + Calendar · Google Maps/Places · Rimrock live availability · Firefly Reservations handoff · Sharp image variants.

**Node version:** 20 (set in `netlify.toml`; install with `nvm install 20`).

**Dev server:** `npm run dev` → Astro dev on port **4321**. See §5 Dev port registry entry below.

**Environment variables (`.env`, gitignored).** Listed by purpose. Real values stay on local machine + Netlify env-var dashboard.
- Supabase: `PUBLIC_SUPABASE_URL`, `PUBLIC_SUPABASE_ANON_KEY`, `SUPABASE_SERVICE_ROLE_KEY`.
- Netlify: `NETLIFY_BUILD_HOOK`.
- Zoho OAuth: `ZOHO_CLIENT_ID`, `ZOHO_CLIENT_SECRET`, `ZOHO_REDIRECT_URI`, `ZOHO_REFRESH_TOKEN`, `ZOHO_ACCOUNT_DOMAIN`.
- Zoho resource IDs: `ZOHO_WORKDRIVE_MEDIA_FOLDER_ID`, `ZOHO_CALENDAR_PUBLIC_EVENTS_ID`.
- Site identity: `SITE_URL`, `ADMIN_EMAIL_FROM`.
- Scheduled-function security: `SCHEDULED_FN_SECRET`.
- Google: `GOOGLE_PLACES_API_KEY` (server-side Place photos/geocoding/static maps), `GOOGLE_MAPS_SERVER_KEY` (server-side Places API New), `PUBLIC_GOOGLE_MAPS_API_KEY` (browser key, referrer-restricted to production + approved dev hosts), and optional `PUBLIC_GOOGLE_MAPS_MAP_ID` (advanced markers; classic markers remain the fallback).

**Active scheduled functions** (`netlify.toml` `[functions.*]`):

**Critical API endpoints to know about:**
- `GET /api/place-photo?name=…&w=…` — server-side proxy for Google Place photos (hides the Places API key).

- `POST /api/zoho/drive-sync` and `POST /api/zoho/calendar-sync` — manual sync triggers (also scheduled).
- `GET /api/sync/status` — admin dashboard pulls this for the System Health panel.
